PATH in Windows: PATH provides absolute paths to executable files. Executable files are run in CMD (Command Prompt). After a path is added to PATH, executables located in that directory can be run by typing their filename without the extension. As a result, commands can be executed from any project directory. (Note: Windows distinguishes different types of content using file extensions, whereas Unix-based operating systems (e.g., Linux) treat everything uniformly as files.)


Command: A command is the filename of an executable file on the operating system. CMD itself can also be considered an executable file, similar to regular software. Example: when opening a .txt file, Windows passes the absolute path of the .txt file as an argument to notepad (the executable/software itself). notepad readsthe file contents via the absolute path and then renders and displays them to the user through its GUI.

When CMD runs a command, it first checks whether the filename contains an absolute path. If it does, CMD searches only in that absolute path for the executable; if not found, it reports an error. -> If the filename does not contain an absolute path, CMD searches the current working directory. -> If still not found, CMD searches built-in OS commands. -> If still not found, CMD searches the directories listed in the PATH environment variable. -> If the executable is not found after all these steps, CMD reports an error.


Heap: The heap is a global storage area. Any thread can allocate memory on the heap. Therefore, heap allocation must be synchronized; otherwise, thread blocking issues may occur. In addition, memory management is required to avoid memory overflow.


Stack and data lifetime: Due to the stack’s first-in, last-out (LIFO) property, the lifetime of a stack frame cannot exceed the scope in which it is called. Data stored on the stack is released when a function returns, so stack data cannot be passed outside the function scope. Therefore, data with a longer lifetime (i.e., data that should persist after leaving a scope) must be stored in a global storage area, namely the heap.


Thread: A thread is the smallest unit of execution scheduled by the operating system. It represents a single control flow during execution. A thread consists of a thread ID, instruction pointer, registers, and a stack. Every program has at least one thread. If a program has only one thread, that thread is the program itself.


Process: A process is composed of threads. A process has its own independent address space, while threads do not have separate address spaces. As a result, threads within the same process share the process’s address space. Multiple processes can execute in parallel, serially, or concurrently.

Windows环境下的PATH：提供可执行文件的绝对路径。可执行文件在CMD（command prompt）中 执行。在PATH中设置路径后，则运行改路径下的可执行文件时，可以直接输入该文件的无后缀名称来运行，因此在任何 工程目录下，命令都可以执行。（注：Windows将各种内容用后缀区分，而基于Unix的操作系统（如Linux）将所有内容统一视为文件。）

命令：操作系统上某个可执行文件的文件名。CMD也可被视为可执行文件，类似软件。例子：打开.txt时 Windows把.txt绝对路径作为参数传给notepad（可执行文件/软件本身）。notepad通过绝对路径读取.txt内容，然后在GUI 界面上渲染并向用户展示。

当CMD运行时，它首先判断文件名是否包含绝对路径。若包含，则只在绝对路径下 寻找对应的可执行文件；若没找到则直接报错。若文件名不包含绝对路径，CMD会在CMD所操作的目录下寻找 可执行文件。若还没找到，CMD会去OS内置命令中找。若还没找到，则去环境变量PATH记载目录中找 可执行文件。以上流程都搜索完后，若还是没有找到，CMD就会报错。

堆：一个全局的存储空间。任何线程都可以在堆上申请空间。因此申请堆内存必须同步处理， 否则会出现线程堵塞的问题。同时，也要进行垃圾对象回收管理，以免内存溢出。

由于栈的first in last out特性， 栈帧生命周期不能超过其被调用的作用域。 栈上数据在函数返回时会被释放，因此栈上原有的数据无法被传输到函数外部。 因此，生命周期较长的数据（指在退出作用域也不消失）只能放到一个全局的存储空间， 也就是堆(heap)上。

线程：执行流课调用的最小单元。为过程中单一的控制流。 由线程ID、指令指针、寄存器、堆栈构成。每个程序都至少由一个线程。若程序 只有一个线程，则其为程序本身。

进程：进程由线程构成。进程由独立的地址空间，但线程没有单独地址空间。 因此，同一进程的线程共享进程的地址空间。多进程可被分为并行、串行、并发等。
